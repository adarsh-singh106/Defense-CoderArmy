Chapter 5: Collecting User Data with Forms

The <form> Element

The <form> element is a container for different types of input elements, such as text fields, checkboxes, radio buttons, submit buttons, etc.

The action attribute defines where the form data is sent when submitted (usually a backend URL), and the method attribute defines the HTTP method (GET or POST).

Key Attributes Explained:

  • label for="...": Connects a label with a specific input field (matches the input's id). Clicking the label will focus the input field.
  • input id="...": A unique ID for the input, used by the label.
  • input name="...": This is crucial! The name is sent to the backend as the key for the data. E.g., username=Rohit.
  • input placeholder="...": Provides a hint to the user about what to enter.
  • input value="...": The actual data associated with an input (especially important for radio buttons and checkboxes).
  • input required: Makes the field mandatory. The form won't submit if it's empty.
