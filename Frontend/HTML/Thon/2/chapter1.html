What is the Internet?

Before diving into web development, we must understand the internet. Think of it as a global network of computers that communicate with each other. The core purpose is communication and data transfer. It's like a giant digital post office system.

The "Post Office" Analogy

Sending data online is very similar to sending a parcel:

  1. The Parcel: This is your data (a message, a photo, a file).
  2. Your Address (Sender): This is your IP Address.
  3. Recipient's Address: This is the destination IP Address.
  4. The Post Office System: This is the network of routers and cables that figures out the best path to deliver your data.

Core Components Explained

1. IP Address (Internet Protocol)

An IP address is a unique numerical label assigned to each device connected to a computer network. It's the "home address" for your device on the internet.

  • IPv4: The older version, like 192.168.1.1. It's a 32-bit number and can support about 4.3 billion devices. We ran out of these!
  • IPv6: The new version, like 2001:0db8:85a3:0000:0000:8a2e:0370:7334. It's a 128-bit number that provides a practically infinite number of addresses.
  • Dynamic IP: Your ISP (Internet Service Provider like Jio, Airtel) gives you a temporary IP address when you connect to the internet. It can change the next time you connect.

2. MAC Address (Media Access Control)

This is the permanent, physical address of your device's network hardware (like your WiFi card). It's burned into the device by the manufacturer and cannot be changed easily. While your IP address is like your temporary home address, the MAC address is like your government ID number - it's uniquely yours.

Why do we need both? If your internet disconnects and your IP is given to someone else (like your dad's phone!), the MAC address ensures that sensitive data meant for you (like Anjali's chat üòâ) doesn't get delivered to him, because the delivery packet checks both the IP and the MAC address.

3. DNS (Domain Name System)

Humans are bad at remembering numbers (IP addresses). We're good with names (like google.com). The DNS is the "phonebook of the internet." When you type a domain name into your browser:

  1. Your computer asks a DNS server, "What is the IP address for google.com?"
  2. The DNS server looks it up and replies, "The IP is 142.250.196.78."
  3. Your browser then connects to that IP address to get the website.

4. Port Numbers

An IP address gets data to the right device, but a port number gets it to the right application on that device. Your computer can run many things at once (Chrome, WhatsApp, Spotify). A port number is like an apartment number in a building.

  • Port 80: For standard web traffic (HTTP).
  • Port 443: For secure web traffic (HTTPS).
  • When you open multiple tabs in Chrome, each tab is assigned a temporary, high-numbered port to keep the data separate.
