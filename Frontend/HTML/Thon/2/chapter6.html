Embedding Media

HTML provides tags to natively embed audio and video content directly into your web pages.

The Audio Tag (<audio>)

Used to embed sound content. The controls attribute is essential to show the play/pause/volume controls to the user.

Best Practice: Use the <source> tag inside the audio tag. This allows you to provide multiple audio formats (e.g., .mp3, .ogg). The browser will use the first one it supports.

The Video Tag (<video>)

Used to embed video content. It has several useful attributes:

  • controls: Shows the default video controls.
  • width/height: Sets the size of the video player.
  • autoplay: Starts the video automatically. Note: Most modern browsers require the muted attribute to be present for autoplay to work.
  • muted: Mutes the audio by default.
  • loop: Makes the video loop back to the beginning when it ends.

Embedding External Content with iFrames

An <iframe> (Inline Frame) is used to embed another HTML document within the current one. This is most commonly used to embed content from other websites, like YouTube videos or Google Maps.

Example: Embedding a YouTube Video

To embed a YouTube video, go to the video, click "Share," then "Embed," and copy the provided iframe code. The URL must be the "embed" version, not the regular "watch" version.

Building a Multi-Page Website

A multi-page website is exactly what it sounds like: a site made of multiple linked HTML files. This entire hackathon project is a multi-page website!

The Core Concept:

  1. Create Separate HTML Files: Each "page" of your site (Home, About, Contact, etc.) is its own .html file. In our project, these are index.html, chapter1.html, etc.
  2. Create a Consistent Navigation: Use a <nav> element on every page. Inside the nav, use anchor tags (<a>) with relative paths to link to your other HTML files.
  3. Maintain a Consistent Structure: Use the same <header> and <footer> across all pages to give your site a unified look and feel.

By clicking the navigation links at the top of this page, you are actively using a multi-page website structure.

Deployment: Making Your Website Live

Deployment is the process of taking your local project files and putting them on a web server so that anyone in the world can access them through a public URL.

How it Works (Simplified):

  1. Gather Your Project Files: This includes all your HTML files, images, videos, and any other assets, all organized in a single folder (like our HACKATHON_PROJECT folder).
  2. Choose a Hosting Platform: There are many free and easy-to-use platforms for static sites (sites made of only HTML, CSS, and JS). Netlify and Vercel are extremely popular.
  3. Upload Your Folder: These platforms often have a simple drag-and-drop interface. You just drag your entire project folder onto their website.
  4. Done! The platform automatically builds and deploys your site, giving you a unique URL (like my-first-project.netlify.app) that you can share with anyone.

That's it! They handle all the complex server configuration, security (HTTPS), and provide a link to your live, multi-page, static website.
